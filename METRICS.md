# 评估指标详细说明

本文档详细解释FEVER事实验证系统中使用的各项评估指标。

## 1. Accuracy（准确率）

### 含义

准确率是最直观的评估指标，表示所有预测中正确预测的比例。

### 计算公式

```
Accuracy = (TP + TN) / (TP + TN + FP + FN)
```

其中：

- TP (True Positive): 真正例 - 预测为正类且实际为正类
- TN (True Negative): 真负例 - 预测为负类且实际为负类
- FP (False Positive): 假正例 - 预测为正类但实际为负类
- FN (False Negative): 假负例 - 预测为负类但实际为正类

### 适用场景

- 类别分布相对平衡时
- 需要整体性能评估时
- 各类别的重要性相同时

### 局限性

- 在类别不平衡时可能产生误导
- 无法反映各类别的具体表现

### 示例

假设有100个样本：

- 正确预测：82个
- 错误预测：18个

```
Accuracy = 82 / 100 = 0.82 (82%)
```

---

## 2. Precision（精确率）

### 含义

精确率衡量模型预测的可靠性，表示预测为某类别的样本中，真正属于该类别的比例。

### 计算公式

```
Precision = TP / (TP + FP)
```

### 含义解释

- 高精确率：模型预测为正类时，大概率是对的（误报少）
- 低精确率：模型预测为正类时，很多是错的（误报多）

### 适用场景

- 关注"预测为正类的准确性"时
- 误报代价高的场景（如垃圾邮件检测）

### 示例

对于"SUPPORTS"类别：

- 模型预测为SUPPORTS：20个
- 其中真正是SUPPORTS：17个

```
Precision = 17 / 20 = 0.85 (85%)
```

解释：模型说"支持"时，有85%的概率是对的。

---

## 3. Recall（召回率）

### 含义

召回率衡量模型的覆盖能力，表示真实为某类别的样本中，被正确预测的比例。

### 计算公式

```
Recall = TP / (TP + FN)
```

### 含义解释

- 高召回率：真正的正类样本，大部分被找出来了（漏报少）
- 低召回率：真正的正类样本，很多被遗漏了（漏报多）

### 适用场景

- 关注"找出所有正类"时
- 漏报代价高的场景（如疾病诊断）

### 示例

对于"SUPPORTS"类别：

- 真实为SUPPORTS：20个
- 模型正确预测出：18个

```
Recall = 18 / 20 = 0.90 (90%)
```

解释：真正"支持"的样本中，90%被模型找出来了。

---

## 4. F1-Score

### 含义

F1-Score是Precision和Recall的调和平均数，综合考虑两者的平衡。

### 计算公式

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

### 为什么用调和平均？

- 算术平均：(P + R) / 2，对极端值不敏感
- 调和平均：更关注较小的值，只有P和R都高时F1才高

### 适用场景

- 需要平衡Precision和Recall时
- 类别不平衡的场景
- 综合评估模型性能时

### 示例

```
Precision = 0.85
Recall = 0.90

F1 = 2 × (0.85 × 0.90) / (0.85 + 0.90)
   = 2 × 0.765 / 1.75
   = 0.874
```

---

## 5. Macro Average（宏平均）

### 含义

对每个类别分别计算指标，然后取平均值。每个类别权重相同。

### 计算公式

```
Macro-Precision = (P1 + P2 + P3) / 3
Macro-Recall = (R1 + R2 + R3) / 3
Macro-F1 = (F1_1 + F1_2 + F1_3) / 3
```

### 适用场景

- 关注每个类别的表现时
- 希望小类别也有同等重要性时

---

## 6. Weighted Average（加权平均）

### 含义

根据每个类别的样本数量加权计算平均值。样本多的类别权重大。

### 计算公式

```
Weighted-Precision = (P1×N1 + P2×N2 + P3×N3) / (N1 + N2 + N3)
```

其中N1, N2, N3是各类别的样本数量。

### 适用场景

- 类别分布不平衡时
- 关注整体性能时

---

## 7. Hallucination Rate（幻觉率）

### 含义

**幻觉率是本项目特有的指标**，用于衡量模型在信息不足时是否倾向于过度自信。

### 定义

真实标签为"NOT ENOUGH INFO"（信息不足）的样本中，模型预测为"SUPPORTS"或"REFUTES"（给出明确判断）的比例。

### 计算公式

```
Hallucination Rate =
    (真实为NOT ENOUGH INFO但预测为SUPPORTS或REFUTES的数量) /
    (真实为NOT ENOUGH INFO的总数)
```

### 含义解释

- **高幻觉率**：模型倾向于"不懂装懂"，在信息不足时仍给出明确判断
- **低幻觉率**：模型比较谨慎，在信息不足时会承认"不知道"

### 为什么重要？

在事实验证任务中：

- 任务一（无证据）：模型可能凭空猜测，幻觉率可能较高
- 任务二（有证据）：有证据支撑，幻觉率应该降低

### 示例

假设有15个真实标签为"NOT ENOUGH INFO"的样本：

- 模型正确预测为"NOT ENOUGH INFO"：12个
- 模型错误预测为"SUPPORTS"：2个
- 模型错误预测为"REFUTES"：1个

```
Hallucination Rate = (2 + 1) / 15 = 0.20 (20%)
```

解释：当真实情况是"信息不足"时，模型有20%的概率会"幻觉"出一个明确答案。

## 8. Confusion Matrix（混淆矩阵）

### 含义

混淆矩阵展示了预测结果与真实标签的对应关系，可以直观看出模型在哪些类别上容易混淆。

### 格式

```
                预测 →
真实 ↓          SUPPORTS    REFUTES    NOT ENOUGH INFO
SUPPORTS            17          2              1
REFUTES              1         12              2
NOT ENOUGH INFO      2          1             12
```

### 解读

- **对角线**：正确预测的数量
- **非对角线**：错误预测的数量
- 可以看出哪些类别容易被混淆

## 指标选择建议

### 任务一（Baseline）

关注指标：

1. **Accuracy** - 整体表现
2. **Hallucination Rate** - 模型是否过度自信
3. **Macro F1** - 各类别平衡表现

### 任务二（RAG）

对比指标：

1. **Accuracy提升** - RAG是否有效
2. **Hallucination Rate降低** - 证据是否减少幻觉
3. **各类别F1变化** - 哪些类别受益最多

---

## 实际应用示例

### 场景1：医疗诊断

- 优先关注：**Recall**（不能漏诊）
- 次要关注：Precision（误诊可以进一步检查）

### 场景2：垃圾邮件过滤

- 优先关注：**Precision**（不能误杀正常邮件）
- 次要关注：Recall（漏掉垃圾邮件影响较小）

### 场景3：事实验证（本项目）

- 平衡关注：**F1-Score**（既要准确又要全面）
- 特别关注：**Hallucination Rate**（避免误导用户）
